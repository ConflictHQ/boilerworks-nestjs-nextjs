# Boilerworks TS — Frontend Bootstrap

Frontend conventions for the Boilerworks TypeScript platform. Agent shims (`CLAUDE.md`, `AGENTS.md`) point here.

An agent given this document and a business requirement should be able to generate correct, idiomatic frontend code without exploring the codebase.

---

## Stack

- Next.js 16 (App Router) + TypeScript
- Apollo Client 4 + `@apollo/client-integration-nextjs`
- Session auth via NestJS backend (httpOnly cookie)
- Tailwind CSS 4 + shadcn/ui
- Prettier + `prettier-plugin-tailwindcss`

---

## Code Formatting

Prettier is configured at the monorepo root: double quotes, semicolons, 2-space indent, ES5 trailing commas. `prettier-plugin-tailwindcss` sorts Tailwind class names automatically.

```bash
npm run format        # rewrite all files
npm run format:check  # CI check (exits 1 if drift)
```

Do not hand-tune whitespace, quote style, trailing commas, or line breaks — Prettier owns those.

## Page Structure

Route groups:

- `app/(app)/` — authenticated pages; layout fetches current user SSR and renders sidebar + header + children
- `app/(login)/` — public; centered container, no sidebar
- Root `app/page.tsx` — redirects: session → `/dashboard`, no session → `/auth/login`

Provider nesting in root layout (outermost → innermost):
`ThemeProvider` → `Toaster` → `ApolloWrapper` → `TooltipProvider`

**Breadcrumbs** are auto-generated by `PageHeader` from `usePathname()`. Each path segment is looked up in `routeLabels` (`lib/routes.ts`); if not found, it falls back to capitalising the segment.

**Adding a new page:**

1. Create `app/(app)/<route>/page.tsx` (Server Component by default)
2. Add a route label in `lib/routes.ts`
3. Add the route to `navMain` in `components/AppSidebar.tsx`

**Page layout boilerplate:**

```tsx
export default function MyPage() {
  return (
    <div className="flex flex-1 flex-col gap-6 p-6">
      <div>
        <h1 className="text-xl font-semibold">Title</h1>
        <p className="text-muted-foreground mt-1 text-sm">Description.</p>
      </div>
      <Separator />
      {/* content */}
    </div>
  );
}
```

## Forms

Stack: `react-hook-form` + `zod` + `@hookform/resolvers/zod`. Forms are always Client Components.

**Pattern:**

1. Define schema in a co-located `schema.ts`; export the inferred type
2. Build form component with `useForm<Values>({ resolver: zodResolver(schema), defaultValues })`
3. Use `register()` for `<Input>` / `<Textarea>`; use `<Controller>` for `<Select>` and other controlled components
4. In `onSubmit`: call mutation, `reset()` on success, `toast.error(...)` on failure
5. Disable submit button with `isSubmitting`

```typescript
// schema.ts
export const mySchema = z.object({
  name: z.string().min(1),
  role: z.enum(["admin", "member"]),
});
export type MyFormValues = z.infer<typeof mySchema>;

// component
const {
  register,
  control,
  handleSubmit,
  formState: { errors, isSubmitting },
  reset,
} = useForm<MyFormValues>({
  resolver: zodResolver(mySchema),
  defaultValues: { name: "", role: "member" },
});
```

## Toasts

Library: `sonner`. The `<Toaster>` is in root layout; theme is synced automatically.

```typescript
import { toast } from "sonner";

toast.success("Saved", { description: "Your changes have been applied." });
toast.error("Failed", { description: "Please try again." });
```

Always provide a title string and an optional `description`.

## Navigation

Sidebar is built from `AppSidebar` (`components/AppSidebar.tsx`). Nav items are configured as plain arrays inside that file.

**`navMain` item shape:**

```typescript
{
  title: string;         // display label
  url: string;           // href
  icon: React.ReactNode; // from lucide-react
  items?: { title: string; url: string }[]; // nested sub-items
}
```

**Adding a nav item:** edit the `navMain` array in `AppSidebar.tsx`. Groups auto-expand when the current path matches any nested URL.

## Data Table

All table primitives live in `components/data-table/`. Library: `@tanstack/react-table` (headless).

### Client-side table

```typescript
"use client";
import { DataTable, DataTableColumnHeader, type FilterConfig } from "@/components/data-table";
import type { ColumnDef } from "@tanstack/react-table";

const columns: ColumnDef<MyRow, unknown>[] = [
  {
    accessorKey: "name",
    header: ({ column }) => <DataTableColumnHeader column={column} title="Name" />,
    enableSorting: true,
  },
];

const filters: FilterConfig[] = [
  { id: "status", label: "Status", type: "select",
    options: [{ value: "active", label: "Active" }] },
  { id: "name", label: "Name", type: "text", placeholder: "e.g. Alice" },
];

<DataTable data={rows} columns={columns} getRowId={(r) => r.id}
           pageSize={10} filters={filters} searchPlaceholder="Search..." />
```

**Key rules:**

- Filter `id` must match `accessorKey`
- Use `DataTableColumnHeader` for all headers — handles sort icon UI
- Pagination and global search are built-in

### Server-side paginated table

For GraphQL-backed tables use `DataTableServer` + `usePaginatedQuery`.

```typescript
const result = usePaginatedQuery<TVariables, TData, TItem, TExtractedData>({
  query, // DocumentNode
  variables, // static partial variables
  ancestor, // key into data — e.g. "users" → data.users.edges / totalCount
  dataExtractor, // (edges: TItem[]) => TExtractedData[]
  storageKey, // localStorage key for persisted perPage preference
  fetchPolicy, // required — e.g. "cache-and-network"
  urlSync, // false | true | "prefix" — syncs pagination to URL params
  filterKeys, // string[] — filter IDs to track
  variablesFromFilters, // (filters) => Partial<TVariables>
  variablesFromSorting, // (sorting) => Partial<TVariables>
});
```

Co-locate columns, filters, and the hook in `graphql/<domain>/<domain>.hooks.tsx`.

## Auth Flow

Session-based. Backend sets an httpOnly cookie on login.

**Frontend auth gate** (`app/(app)/layout.tsx`):

- Server Component — checks for session cookie
- If no cookie → redirect to `/auth/login`
- If cookie → fetch user via GraphQL → render children

**Key files:**

```
lib/auth.ts         — getSession(), requireAuth() (server-only)
lib/auth-client.ts  — useCurrentUser() hook (client)
```

## Apollo Setup

Two clients — one for server (RSC), one for client components.

**Server client** (`lib/apollo/client.ts`):

- Uses `registerApolloClient` from `@apollo/client-integration-nextjs`
- Reads session cookie for auth
- Exports `getClient`, `query`, `PreloadQuery`

**Client provider** (`lib/apollo/provider.tsx`):

- `"use client"` — wraps app in `ApolloNextAppProvider`
- Token injected per-request via auth link
- Includes `errorLink`: redirects to `/auth/login` on `UNAUTHENTICATED`

**Cache** (`lib/apollo/cache.ts`):

- `InMemoryCache` from `@apollo/client-integration-nextjs` (required for SSR)
- `typePolicies` defined explicitly — no Proxy catch-all
- Add `merge: true` per type as needed

**Public API** — always import from the barrel:

```typescript
import { getClient, PreloadQuery, ApolloWrapper } from "@/lib/apollo";
```

## GraphQL Operations — File Structure

All operations live in `graphql/<domain>/`:

```
graphql/
  user/
    user.fragments.ts   — gql fragments
    user.queries.ts     — DocumentNode exports (GET_ME, etc.)
    user.mutations.ts   — mutation DocumentNode exports
    user.hooks.ts       — arrow-function hooks wrapping useQuery/useMutation
    user.types.ts       — manual TS types (replaced by codegen later)
```

When adding a new domain (e.g. `orders`), create `graphql/orders/` with the same 5 files.

## Adding a New Query

**1. Define types** in `<domain>.types.ts`:

```typescript
export type MyQueryData = { thing: { id: string; name: string } | null };
export type MyQueryVariables = { id: string };
```

**2. Write the query** in `<domain>.queries.ts`:

```typescript
export const GET_THING = gql`
  query GetThing($id: ID!) {
    thing(id: $id) {
      id
      name
    }
  }
`;
```

**3. Add a hook** in `<domain>.hooks.ts`:

```typescript
export const useThing = (id: string) => {
  const { data, loading, error } = useQuery<MyQueryData, MyQueryVariables>(GET_THING, {
    variables: { id },
    fetchPolicy: "cache-first",
  });
  return { thing: data?.thing ?? null, loading, error };
};
```

**4. Use in a client component:**

```typescript
"use client";
import { useThing } from "@/graphql/user/user.hooks";
```

**5. Use in a server component (RSC):**

```typescript
import { getClient } from "@/lib/apollo";
import { GET_THING } from "@/graphql/user/user.queries";

const client = await getClient();
const { data } = await client.query({ query: GET_THING, variables: { id } });
```

## Adding a Mutation

**1. Write the mutation** in `<domain>.mutations.ts`:

```typescript
export const UPDATE_THING = gql`
  mutation UpdateThing($input: UpdateThingInput!) {
    updateThing(input: $input) {
      ok
      errors {
        field
        message
      }
    }
  }
`;
```

**2. Add a hook** in `<domain>.hooks.ts`:

```typescript
export const useUpdateThing = () =>
  useMutation<UpdateThingData, UpdateThingVariables>(UPDATE_THING, {
    refetchQueries: [GET_THING],
  });
```

## Fragments

Fragments live in `<domain>.fragments.ts`. Confirm exact type names from the backend schema — wrong names return a 400.

```typescript
export const THING_FIELDS = gql`
  fragment ThingFields on Thing {
    id
    name
  }
`;

export const GET_THING = gql`
  query GetThing($id: ID!) {
    thing(id: $id) {
      ...ThingFields
    }
  }
  ${THING_FIELDS}
`;
```

## RSC vs Client Component Rules

|               | Server Component               | Client Component                           |
| ------------- | ------------------------------ | ------------------------------------------ |
| Data fetching | `await getClient().query(...)` | `useQuery` hook via `<domain>.hooks.ts`    |
| Mutations     | not applicable                 | `useMutation` hook via `<domain>.hooks.ts` |
| Auth state    | `getSession()` from `lib/auth` | `useCurrentUser()` hook                    |

- Never use `useQuery` / `useMutation` in a Server Component
- Never use `getClient()` in a Client Component — use hooks instead
- Always import hooks from `@/graphql/<domain>/<domain>.hooks`, not `useQuery` directly

## Permission Guards

Two complementary patterns: server guard (RSC) and client guard.

**Key files:**

```
graphql/permissions/
  permissions.types.ts    — PermissionSlug enum (source of truth)
  permissions.queries.ts  — GET_PERMISSION query
  permissions.hooks.ts    — usePermission(slug) hook (client)
lib/permissions.ts        — checkPermission / requirePermission (server-only)
components/PermissionGuard.tsx — PermissionGuard + HOC (client)
```

### Server guard (RSC pages)

```tsx
import { requirePermission, PermissionSlug } from "@/lib/permissions";

export default async function MyPage() {
  await requirePermission(PermissionSlug.MyFeature);
  return <div>...</div>;
}
```

### Client guard

```tsx
import { PermissionGuard } from "@/components/PermissionGuard";
import { PermissionSlug } from "@/graphql/permissions/permissions.types";

<PermissionGuard permission={PermissionSlug.MyFeature}>
  <MyContent />
</PermissionGuard>;
```

## Conventions

- Code style is enforced by Prettier — do not hand-tune formatting
- Hooks use **arrow functions**: `export const useMe = () => { ... }`
- Query/mutation constants: `SCREAMING_SNAKE_CASE` (`GET_ME`, `UPDATE_PROFILE`)
- Paginated list queries: `camelCase` (`getEmployees`) — used with `usePaginatedQuery`
- Fragment names: `PascalCase` + `Fields` suffix (`ThingFields`)
- All `useQuery` calls must have explicit type params: `useQuery<Data, Variables>(...)`
- `fetchPolicy` must be set explicitly in every hook — no implicit defaults
- Component files: **PascalCase** (`AppSidebar.tsx`). Files with JSX columns use `.tsx` (`employees.hooks.tsx`)

## Utility Hooks

All utility hooks live in `hooks/`. Import directly:

```typescript
import { useDebounce } from "@/hooks/use-debounce";
import { useLocalStorage } from "@/hooks/use-local-storage";
import { useCopyToClipboard } from "@/hooks/use-copy-to-clipboard";
import { useConfirm } from "@/hooks/use-confirm";
```

### useDebounce

```typescript
const debouncedSearch = useDebounce(searchValue, 300);
```

### useLocalStorage

```typescript
const [theme, setTheme] = useLocalStorage<string>("theme", "system");
```

SSR-safe typed wrapper around `localStorage`.

### useCopyToClipboard

```typescript
const [copied, copy] = useCopyToClipboard();
```

### useConfirm

Requires `<ConfirmProvider>` in the tree (already in root layout).

```typescript
const confirm = useConfirm();
const ok = await confirm({
  title: "Delete item?",
  description: "This cannot be undone.",
  confirmLabel: "Delete",
  cancelLabel: "Cancel",
});
if (ok) deleteItem();
```
